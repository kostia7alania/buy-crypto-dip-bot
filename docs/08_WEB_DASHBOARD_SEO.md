# Web Dashboard and SEO

## Framework

Nuxt 4.

## Why Nuxt

The product needs SEO route experiments. Use SSR/SSG pages, not a plain client-side SPA.

## Dashboard

Risk-first layout:

1. Mode: DRY_RUN / LIVE
2. Risk status
3. Daily spend
4. Weekly spend
5. Last BTC price
6. Last signal
7. Rejected signals
8. Dry-run orders
9. Audit log
10. Telegram bot status

## Design rules

- boring
- readable
- no casino UI
- no huge green buy button
- no PnL dopamine screen
- no meme coins
